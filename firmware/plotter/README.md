# plotter — main firmware

Drives the full plotter (XY steppers + pen servo) and exposes a small HTTP REST API on the local network.

## Endpoints

mDNS hostname: **`http://drawbot.local`** (Bonjour required on Windows; built into macOS/iOS/most Linux).

| Method | Path | Body | Returns |
|---|---|---|---|
| `GET` | `/status` | — | JSON: `{state, pen, x_mm, y_mm, queue_remaining, last_error}` |
| `POST` | `/job` | `text/plain` G-code (multi-line) | `202 {queued: N}` or `409 busy` or `400 parse error` |
| `POST` | `/stop` | — | aborts current job, lifts pen |
| `POST` | `/home` | — | runs `G28` (drive to 0,0) |

## G-code subset

| Code | Meaning |
|---|---|
| `G0 X.. Y..` | rapid move (caller decides pen-up convention) |
| `G1 X.. Y..` | linear move |
| `G28` | home (drive to 0,0) |
| `G90` / `G91` | absolute / relative coordinates |
| `M3` / `M5` | pen down / pen up |

Unsupported codes return `400 parse error`. Comments (`;` or `(...)`) are stripped.

## Architecture

```
firmware/plotter/
  plotter.ino       → setup() + loop() — composition root
  config.h          → pins, speeds, build envelope, network names
  gcode.{h,cpp}     → text → Command struct
  motion.{h,cpp}    → XY + servo, job queue, non-blocking state machine
  http_api.{h,cpp}  → WebServer routes (UI + JSON API)
  wifi_setup.{h,cpp}→ WiFiManager captive portal + mDNS
  web_assets.h      → generated by scripts/build_web.sh — gitignored

/web/               → Vite + Svelte SPA (source)         (separate folder)
/scripts/build_web.sh → glue: web/dist → web_assets.h    (separate folder)
```

The SPA source lives at the **repo root** under `/web/` — it's a separate concern from the firmware and has its own toolchain. See `/web/README.md` for the dev workflow.

## Build + flash (from repo root)

```bash
( cd web && npm install && npm run build )    # first time: install + build SPA
./scripts/build_web.sh                         # bake into firmware/plotter/web_assets.h
arduino-cli compile --fqbn esp32:esp32:esp32 firmware/plotter
arduino-cli upload  --fqbn esp32:esp32:esp32 -p /dev/cu.usbserial-0001 firmware/plotter
```

`vite-plugin-singlefile` inlines all JS/CSS into one HTML so the firmware only needs to serve a single asset at `/`. Bundle is ~14 KB raw, ~5.5 KB gzipped.

The motion module never blocks. `motion_tick()` advances a state machine; `xyGroup.run()` returns `false` when the active move completes, which triggers the next command in the queue. This keeps the HTTP server responsive while the plotter is drawing.

## Flash

```bash
arduino-cli compile --fqbn esp32:esp32:esp32 firmware/plotter
arduino-cli upload  --fqbn esp32:esp32:esp32 -p /dev/cu.usbserial-0001 firmware/plotter
arduino-cli monitor -p /dev/cu.usbserial-0001 -c baudrate=115200
```

## Wi-Fi setup — two modes

**Default (recommended for tutorial readers): captive portal.**

1. After flashing, watch the serial monitor.
2. The ESP32 hosts an open Wi-Fi network called **`TinyRoboCo-Setup`**.
3. From your phone or laptop, connect to that network.
4. A captive portal opens automatically (or visit `http://192.168.4.1`).
5. Tap **Configure WiFi**, pick your home network, enter its password, save.
6. ESP32 reboots, connects to your network, and answers at `http://drawbot.local`.

Credentials are stored in the ESP32's NVS flash, not in the firmware. One-time setup.

**Optional: hardcoded credentials (skip the captive portal).**

If you're iterating on firmware and don't want to re-do the portal dance after each flash:

```bash
cp firmware/plotter/secrets.example.h firmware/plotter/secrets.h
# edit secrets.h with your SSID / password
```

`secrets.h` is in `.gitignore` — your credentials never enter the repo. The presence of `secrets.h` automatically swaps in a direct-connect path at compile time and drops the WiFiManager dependency.

## Try it

```bash
# status
curl http://drawbot.local/status

# draw a 50mm square (no pen, just motion test)
curl -X POST http://drawbot.local/job \
  -H 'Content-Type: text/plain' \
  --data-binary $'G90\nG0 X0 Y0\nG1 X50 Y0\nG1 X50 Y50\nG1 X0 Y50\nG1 X0 Y0\n'

# home
curl -X POST http://drawbot.local/home

# emergency stop
curl -X POST http://drawbot.local/stop
```

## Security note

The API is **unauthenticated** and accessible to anything on your LAN. Fine for a home network. Before exposing to the internet, add a token check or a reverse proxy with auth in front. That's a future tutorial chapter.
